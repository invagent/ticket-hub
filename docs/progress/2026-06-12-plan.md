## 三、后续计划（重排后）

### 第 1 段：Linear 闭环 + 主管运营体验（~1.5w，最高优先）

做完这段，Bug_fix/Demand 的完整生命周期（进 → 推 → 回）就闭环了：

1. **Linear 状态回同步**（D4 余项）
   - 方案：Celery beat 轮询（5min 扫 `linear_uuid` 非空且未完结的 hub_issue → GraphQL 查 state → 回写 `linear_status`）；webhook 方案等有公网回调需求再升级
   - 状态映射落 `status_cascade` 雏形：Linear Done → hub_issue `released`/`resolved`
2. **主管工作台运营 UI 补全**
   - `status=pending` 队列（筛选 + 「重推 Linear」按钮）
   - dedup 提案卡片（复用拆单卡片模式：链接/忽略）
   - 「从 Linear 同步用户」按钮（管理后台用户页）
3. **灰度推进**：基于积累的 agent_decisions 审计数据评估命中率 → 达标后开 `HUB_ISSUE_AUTO_ENABLED`（保持 0.80 阈值），dedup/split 自动化继续观察

6. 前端 `/hub-issues` 4 出口类型分视图（核心运营页）
